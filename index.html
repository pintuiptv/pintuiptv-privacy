Dati trattati

  • Dati tecnici (es. log di errore e informazioni sul dispositivo) solo se necessari per funzionamento/diagnostica.
  • Dati di utilizzo (es. preferiti, cronologia, impostazioni, progressi di visione) salvati localmente sul dispositivo. Per impostazione predefinita questi dati sono visibili solo all’utente e non al titolare dell’applicazione.
  • Dati forniti dall’utente (es. liste/URL/credenziali) utilizzati esclusivamente per erogare le funzionalità dell’applicazione. Per impostazione predefinita restano sul dispositivo e sono visibili solo all’utente.
  • Dati di sincronizzazione cloud (solo se l’utente attiva la funzione di sincronizzazione): una copia di alcuni dati di utilizzo può essere caricata su cloud per consentire backup/ripristino e continuità tra dispositivi.

Sincronizzazione Cloud (Firebase)

Se l’utente attiva la sincronizzazione, l’app utilizza servizi Firebase (in particolare Cloud Firestore) per memorizzare una copia dei dati necessari alla sincronizzazione tra dispositivi.

  • Cosa può essere sincronizzato (a seconda delle funzionalità attivate nell’app): preferiti, stati di visione/progresso, cronologia, impostazioni e altri dati di utilizzo.
  • Identificatore di sincronizzazione: può essere utilizzato un codice/ID scelto o generato dall’app per collegare i dati al profilo di sync dell’utente.
  • Finalità: backup, ripristino, continuità di utilizzo su più dispositivi.
  • Base giuridica: consenso dell’utente, espresso tramite l’attivazione della funzione di sincronizzazione (e disattivabile in qualunque momento).

Finalità

  • Fornire e mantenere le funzionalità dell’app.
  • Assistenza e risoluzione problemi.
  • Sicurezza e prevenzione abusi.
  • Backup e sincronizzazione tra dispositivi (solo se attivata dall’utente).

Condivisione dei dati

I dati non vengono venduti. I dati locali restano sul dispositivo dell’utente. Se l’utente attiva la sincronizzazione cloud, i dati di sincronizzazione vengono trattati tramite servizi Firebase secondo le relative informative. Eventuali altri fornitori terzi (es. crash reporting/analytics) sono utilizzati solo se presenti/attivati e secondo le rispettive informative.

Conservazione

  • Dati locali: restano sul dispositivo finché l’utente non li elimina o disinstalla l’app.
  • Dati su cloud (Firebase): restano disponibili per la sincronizzazione finché l’utente mantiene attiva la funzione oppure finché non richiede la cancellazione. La disattivazione della sincronizzazione o la rigenerazione/azzeramento dell’ID di sync (se previsto dall’app) può interrompere l’associazione ai dati precedentemente caricati.

Sicurezza

I dati sono trasmessi tramite connessioni protette (HTTPS/TLS). I servizi cloud utilizzati (se attivati) applicano misure di sicurezza previste dal fornitore, incluse protezioni in transito e a riposo.

Diritti dell’utente

L’utente può richiedere informazioni e assistenza sulla gestione dei dati, inclusa l’eventuale cancellazione dei dati di sincronizzazione, contattando: pintuiptv.help@outlook.com

Data Controller

Pintu IPTV
Contact: pintuiptv.help@outlook.com

Data We Process

  • Technical data (e.g., error logs and device information) only when needed for app operation and troubleshooting.
  • Usage data (e.g., favorites, history, settings, watch progress) stored locally on the device. By default, this data is visible only to the user and not to the app owner.
  • User-provided data (e.g., playlists/URLs/credentials) used only to provide the app’s features. By default, this data remains on the device and is visible only to the user.
  • Cloud sync data (only if the user enables sync): a copy of certain usage data may be uploaded to the cloud to enable backup/restore and continuity across devices.

Cloud Sync (Firebase)

If the user enables synchronization, the app uses Firebase services (in particular Cloud Firestore) to store a copy of the data required to sync across devices.

  • What may be synced (depending on the features enabled in the app): favorites, watch state/progress, history, settings, and other usage data.
  • Sync identifier: the app may use a user-provided or app-generated code/ID to link data to the user’s sync profile.
  • Purpose: backup, restore, and seamless use across devices.
  • Legal basis: user consent, provided by enabling the sync feature (which can be disabled at any time).

Purposes

  • Provide and maintain the app’s functionality.
  • Support and troubleshooting.
  • Security and abuse prevention.
  • Backup and cross-device synchronization (only if enabled by the user).

Data Sharing

Data is not sold. Local data remains on the user’s device. If cloud sync is enabled, synced data is processed using Firebase services under their applicable policies. Any additional third-party providers (e.g., crash reporting/analytics) are used only if present/enabled and under their respective policies.

Retention

  • Local data: remains on the device until the user deletes it or uninstalls the app.
  • Cloud data (Firebase): remains available for synchronization while the user keeps the feature enabled or until the user requests deletion. Disabling sync or resetting/regenerating the sync ID (if supported by the app) may break the association with previously uploaded data.

Security

Data is transmitted via secure connections (HTTPS/TLS). If enabled, cloud services apply security measures provided by the vendor, including protections in transit and at rest.

User Rights

Users can request information and support regarding data management, including deletion of synced data, by contacting: pintuiptv.help@outlook.com
